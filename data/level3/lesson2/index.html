数组所占内存大小

（2023年9月）在C++语⾔中，数组下标的大小决定元素在逻辑上的先后顺序，与元素在内存中位置的先后顺序无关。
A 正确
B 错误
【解析】数组在内存中是连续存储的，下标越小的元素在内存中位置越靠前。数组下标决定元素在逻辑和物理上的先后顺序。
（样题）C++语言中的数组可以根据需要自动调整大小。
A 正确
B 错误
【解析】C++语言中的数组大小在定义时确定，一旦定义不能自动调整大小。如果需要动态调整大小，应使用vector等容器。
（2023年6月）一个数组定义为 double array[3];，则这个数组占用内存的大小为（ ）。
A 24
B 12
C 6
D 3
【解析】double类型占用8字节，数组有3个元素，所以总大小为3×8=24字节。
（2023年9月）如果数组定义为 long long array[] = {3, 5, 7, 2}; ， 则数组 array 占用的字节数为（ ）。
A 32
B 16
C 8
D 4
【解析】long long类型占用8字节，数组有4个元素，所以总大小为4×8=32字节。

定义数组的位置

（2023年9月）在C++语⾔中，长度为n的数组，访问下标为n的元素会引起编译错误。
A 正确
B 错误
【解析】数组下标越界是运行时错误，不会引起编译错误。C++编译器不检查数组下标是否越界。
（2023年6月）下列关于 C++语言中数组的叙述，不正确的是（ ）。
A 数组必须先定义后使用。
B 数组的所有元素在内存中是连续存放的。
C 除了字符数组，在定义数组时“[]”内必须有常数。
D 不能对数组赋值，但可以对数组的每个基础类型的元素赋值。
【解析】选项C不正确。任何类型的数组在定义时都可以省略长度，但必须提供初始值列表，不只是字符数组。
（2023年9月）下列关于C++语言中数组的叙述，不正确的是（ ）。
A 可以定义 0 个元素的数组
B 不能定义 -1 个元素的数组
C 数组下标越界访问会产生编译错误。
D 程序运行时发生数组下标的越界访问，程序依然可能正常结束。
【解析】选项C不正确。数组下标越界是运行时错误，不会产生编译错误。

遍历数组元素

（2023年6月）在下列代码的横线处填写（ ），可以使得输出是“2”。
A min > array[i]
B min < array[i]
C min = array[i]
D 以上均不对。
【解析】从代码可以看出，数组初始值为{3, 7, 7, 5, 2, 4}，min初始化为0。如果条件是 min > array[i]，当 min=0 时，所有元素都大于0，都会执行赋值，最终min=4；如果条件是 min < array[i]，则所有元素都大于0，最终min=4；如果条件是 min = array[i]，这不是比较操作而是赋值，语法有误。因此正确答案是D。
（2023年9月）在下列代码的横线处填写（ ），可以使得输出是“120”。
A res += array[i];
B res *= array[i]
C res = array[i]
D 以上均不对。
【解析】从代码可知数组元素为{1, 2, 3, 4, 5}，初始res=0。选项A累加结果为15，选项B累乘结果为0（0乘以任何数都是0），选项C最后赋值为5。三个选项都无法输出120，所以答案选D。
（2023年9月）在下列代码的输出是（ ）。
A 15
B 28
C 45
D 55
【解析】从代码可知数组元素为{0, 1, 1, 2, 2, 4, 2, 6, 4, 6}，程序对数组元素进行累加求和。0+1+1+2+2+4+2+6+4+6=28，所以答案选B。
