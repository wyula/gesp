（样题）C++语言中，如果异常发生，但没有处理异常的代码，则程序会由于一直等待处理而死机。
A 正确
B 错误
【解析】本题考查异常处理的概念。如果异常发生但没有处理异常的代码，程序会直接终止运行，而不是一直等待处理。
课堂练习 异常是指（ ）
A 程序设计时的错误
B 程序编写时的错误
C 程序编译时的错误
D 程序运行时的错误
【解析】本题考查异常的定义。异常是指程序在运行过程中发生的错误，如除数为0、内存分配失败、文件不存在等。
（样题）在 C++中，以下哪个关键字用来捕获异常（ ）
A throw
B catch
C try
D finally
【解析】本题考查异常处理关键字。try用于包裹可能发生异常的代码，catch用于捕获异常，throw用于抛出异常。C++中没有finally关键字（Java中有）。
课堂练习 try与catch总是结合使用的。
A 正确
B 错误
【解析】本题考查try-catch的用法。try必须和catch配合使用，单独的try或catch都是语法错误。
（2023年9月）下列关于C++语言中异常处理的叙述，正确的是（ ）
A 一个 try 子句可以有多个 catch 子句与之对应
B 如果 try 子句在执行时发生异常，就一定会进入某一个 catch子句执行
C 如果 try 子句中没有可能发生异常的语句，会产生编译错误
D catch 子句处理异常后，会重新执行与之对应的 try 子句
【解析】本题考查异常处理的特性。一个try子句可以对应多个catch子句，分别处理不同类型的异常。选项B错误，因为如果没有匹配的catch，异常会继续向上传播；选项C错误，try块中可以没有异常语句；选项D错误，catch处理完后不会重新执行try。
（2023年6月）在 C++语言中，如果一个函数可能抛出异常，那么一定要在 try 子句里调用这个函数。
A 正确
B 错误
【解析】本题考查异常处理的使用。函数可能抛出异常，但不一定要在try子句中调用。如果不在try中调用且发生异常，异常会向上传播，直到被捕获或程序终止。
（2023年9月）一个可能抛出异常的函数，调用它的位置没有在 try 子句中，会引起编译错误。
A 正确
B 错误
【解析】本题考查异常处理的规则。在C++中，调用可能抛出异常的函数不需要必须在try子句中，这不会引起编译错误。只是如果发生异常且没有被捕获，程序会终止。
课堂练习 如果某段程序中发现了自己不能处理的异常，就可以使用throw<表达式>抛掷这个异常，其中的<表达式>表示异常类型。
A 正确
B 错误
【解析】本题考查throw语句的用法。throw后面的表达式的值和类型决定了抛出的异常，catch根据类型进行匹配。
课堂练习 C++的异常处理机制使得异常的引发和处理不必在同一函数中。
A 正确
B 错误
【解析】本题考查异常处理机制。C++的异常处理采用栈展开机制，异常可以从被调用函数抛出，由调用链上层的try-catch捕获处理。
课堂练习 下列关于异常处理的流程的描述中，错误的是（ ）
A 对某段可能产生异常的代码或函数使用try结构进行检测
B 如果在执行try结构期间没有引起异常，则跟在try后面的 catch结构不会执行
C 如果在执行try结构期间发生异常，则在异常发生的位置 使用throw抛出异常，一个异常对象将被创建
D 本层try语句抛出了异常，只能由本层的catch语句处理
【解析】本题考查异常处理流程。选项D错误，因为异常可以向上传播，如果本层没有匹配的catch，异常会继续向上层调用栈传播，直到被捕获或程序终止。
